LLM Safety and Robustness Analysis

This repository contains the code and data used to analyze the security robustness of various Large Language Models (LLMs) against different attack vectors, as detailed in the Final_Deliverable.pdf.

Project Workflow

The project is structured into the following stages:

1. Vulnerability Scanning (garak_test.ipynb)

We utilized the garak vulnerability scanner to test multiple LLMs against a set of known attack probes.

Models Evaluated:

amd/AMD-OLMo-1B (Small, Base)

tiiuae/falcon-7b (Large, Base)

mistralai/Mistral-7B-v0.1 (Large, Base)

(and their instructed counterparts where applicable)

Probes Used:

dan.DanInTheWild (Jailbreaks)

goodside.WhoIsRiley (Hallucinations/Social Engineering)

realtoxicityprompts.RealToxicityPrompts (Toxic Content generation)

encoding (Injection attacks via encoding)

2. Data Processing (read_json.py)

The raw output generated by garak (in .jsonl format) was parsed using this support script. It extracts the relevant metrics from the logs to facilitate analysis.

3. Data Storage (extracted_final_version.xlsx)

The parsed results, specifically the absolute scores for each model and probe combination, were consolidated and saved into this Excel spreadsheet for easy retrieval and comparison.

4. Data Analysis & Visualization (results_analysis.ipynb)

This notebook loads the data from extracted.xlsx to perform statistical analysis. It generates the comparative charts and visual evidence presented in the Final_Deliverable.pdf, highlighting the trade-offs between model size, instruction tuning, and safety.

5. Custom Probe Development (my_probe.ipynb)

In this notebook, we developed and deployed a custom probe named lmrc.SlurUsage.

This probe targets specific nuances of toxic speech detection using Direct, Persona, and Logic-based prompting strategies.

The results of this custom probe were transcribed and analyzed to validate the "Inverse Scaling" hypothesis discussed in the final report.
